The tree of the user's personal mobility. It is composed by two prefix trees starting from a unique location: one encodes the paths arriving to it, the other the paths exiting from the root location.

A prefix tree is a particular structure that groups together the travels that share the same initial path.

One location can then appear in different branches because it can occour after different sequences of traversed points.

The trajectories entering the root (the central point) can continue and become outcoming paths.

The same representation of the mobility prefix tree but mapped on the distance traveled.

For each depth level of the incoming and the outcoming tree, it represents which percentage of all the leaf locations are in that level.

The median and the maximum number of successor that a location node can have.

This information is useful to understand how much fragmented is the user's mobility and how much broad are the hubs (that is locations with many possible following directions).

The location richness of a tree is the proportion between the number of all the points traversed and the number of different locations encountered.

It gives information about how much habitudinary is the user: if the richness is low, in fact, it means that the same locations are traversed quite frequently; on the contrary, an elevated richness means that the user explores many different locations, and rarely returns on the same routes.

The distribution of the entropies of each location.

The entropy is the amount of byte necessary to describe a location, according to Claude-Shannon's information theory.
